#railsconf 2014 roundup.

23 Apr 2014

Last week I had grand plaisir of joining my colleagues from Science Exchange at Rails Conf in Chicago. Hella good conference! Especially for a front-sider turning full stacks racker. Man I was hungering for that deep dish wisdom of the backends. I was not disappointed.

Probs the biggest takeaway is that no question remains: an understanding of the full web stack is crucial for any contemporary front-ender building a software application. For whatever flavor of front-side craftgineer you are: UI eng, UX eng, Front-end Eng, JavaScript Eng... We must be strapped up and down straight Punisher ish. I mean, this was always already true - but, if you're like me, maybe your career is young and you've been mostly scripting and styling DOM states with jQuery mixed with sprinkles of AJAX for sporadic sexy data fetching. Maybe that could get you far in the past. But the web stack is growing up fast, exponentially, and apps are going to have that familiar JavaScript feel, they're gonna look like Rails apps, or Django apps.

Any moment now you're going to be sitting at the terminal looking at codes called Lineman, Grunt, Gulp, Broccoli, maybe some thing called Perspectives Rap Genius is building, Node, Handlebars, Ember, Angular, Rendr, Backbone, Yoeman, D3, and we haven't started talking TDD yet. You'll be required to understand MVC, hexagons and domains, or some similar architectural pattern to deliver that a juicy SPA experience to users who don't want no static. You must become a veritable master of state. I'm not sure what the winning stack is these days, but an EmberJS client backed with a Rails JSON API was a hot combo at Rails conf - for obvious reasons (read: Yehuda).

Ember is picking up speed. Go watch Brandon Hays' talk and start sprinkling in Ember right now. Agile that shit right in there.

Ok, after that long intro here's the conference - distilled - from my front-side perspective:

  • Ruby/Rails programmers hate JavaScript.
  • Rails programmers are going to be OK because a Rails guy built a JavaScript client framework called Ember.
  • Now there's a JavaScript asset pipeline called Broccoli that's way better than Grunt.
