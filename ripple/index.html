( Zendesk and me. 2010 - 2013. )

Hi Ripple. Here's some example work from my time at Zendesk.

Benchmark Page

zendesk.com/benchmark-your-support

After being handed a flat file from Creative, I spruced up the live page with an onload animation to "rev-up" the user, as well as Vegas-ify the Benchmark data table with a slot-machine effect. The data from this page is retrieved by an AJAX call using the Google Data API which fetches data from a Google Spreadsheet - not the best data-store but a solution at the time because that's where the Product data team put it. Also, it's subtle, but the Global Benchmark numbers are dynamic data so the numerals are actually loaded from a sprite.

Internationalization

  • ja.zendesk.com
  • de.zendesk.com
  • fr.zendesk.com

    • You'll notice design/UI parity on most landing pages across our English site and localized sites. This is made possible by leveraging native WordPress frameworks and APIs like multi-site network, child themes, and the GNU gettext utilities baked into WordPress core. By abstracting our translation files form our templates we were able to roll out new landing pages to our localized sites rapidly and even train less technical team members to make translation edits.

    Sitewide Search

    zendesk.com

    In an effor to provide the user with a simpler way to get information quickly about our product, I redesigned and developed our sitewide search to be unobtrusively accessible from the top navigation. It can also be toggled by typing the "s" key. I added autocomplete which does regex matching with a jQuery plugin from a common index of terms researched from historical searches reported in Google Analytics. Results are retrieved from an AJAX call to the Google Search Engine API on submit.

    Apps

    zendesk.com/apps

    Creative was swamped so I designed the entire UI for the single app pages - click "Learn More" on any tile to view. Underneath the hood there is actually a custom template routing plugin I wrote in PHP that dynamically builds the page based on the URI request - these pages are not stored in the MySQL database because the data comes from the Zendesk product! Data is actually stored in a JSON file that we update periodically with a Grunt.js task during development.

    BONUS: CSS/JS Minification and Cache Busting (under the hood)

    In the source code you'll notice hashes and a ".min" suffix on our externally-loaded CSS and JavaScript files. I implemented a set of Grunt.js tasks to minify and hash files, and then make replacements throughout our WordPress templates.
