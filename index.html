It was no small feat integrating these new data models into heavily indebted codebases. I'm looking forward to working with another team of curious learners at a medium to large company on similarly impactful core products.

Throughout my career I've frequently touched server code (Ruby on Rails, Django, SQL) and have been regularly involved in API design. I've also mentored in microservice development and Python.

At Eventbrite I worked in dozens of JavaScript codebases and received peer mentorship from some of the best in the business. I conquered a new stack (Backbone, React, the Flux pattern, Redux) and deepened my expertise in event-driven software to become a top contributor to new feature development and code quality efforts across the organization. The latter endeavor was unique and became a veritable master class in appropriately calibrating design consistency and autonomy across geos for a rapidly evolving system of people, process, and software.

These challenges inspired a lot of personal growth through technical investigations along a few different pillars of inquiry/implementation that I'm proud of.

Architecture: Orienting our React code toward control patterns like finite state machines and pull-vs-push to clarify abstractions to enforce UI = fn(state), as well as socializing API documentation and shared data structures for service developers as cardinal contracts to increase flow for frontend engineers.

Communications: Inventing emoji-based semantics for Slack comms, credentialed training in Nonviolent Communication, and encoding decision-making through architectural design records (I'd really like to see ADRs extended more into version control alongside code artifacts!) and bi-monthly working groups.

Code: Expanding our toolset to meet the challenges of consistency and rigor, the boldest choice of which was embracing a static type system -- TypeScript -- and pioneering typing in our organization, as well as clarifying and establishing principles for "functional strength" in modules based on first principles (partially inspired by introductions to the FORTH programming language).
