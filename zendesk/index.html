Hi Ripple. Here are a few examples of my work at Zendesk.

The Zendesk Benchmark Page

zendesk.com/benchmark-your-support | gist

After being handed a flat file from Creative, I spruced up this page in the browser with an onload animation to "rev-up" the user, as well as Vegas-ify the Benchmark data table with a slot-machine effect. The data displayed on the page is fetched by an AJAX call from a Google Spreadsheet (Google Data API) - not the best data-store but a stop-gap solution at the time because that's where the Product data team put it. Also, it's subtle, but the Global Benchmark numbers are dynamic data so the numerals are actually loaded from a sprite.

Internationalization

  • ja.zendesk.com
  • de.zendesk.com
  • fr.zendesk.com

    • You'll notice design/UI parity on most landing pages across our English site and localized sites. This is made possible by leveraging native WordPress frameworks and APIs like the multi-site network, child themes, and the GNU gettext libraries baked into WordPress core. By using gettext we were able to abstract our translation files from our templates, which allowed the team to roll out new landing pages to our localized sites rapidly, and even train less technical team members to make translation edits in GUI tools for Mac.

    Sitewide Search

    zendesk.com | gist

    Zendesk is a product used by a wide spectrum of industries and company types. The feature set and benefits are complex so the messaging is fractured across dozens of landing pages on the corporate website. In an effort to provide the user with a simpler way to get information quickly about our product, I redesigned and developed our sitewide search to be unobtrusively accessible from the top navigation. It can also be toggled by typing the "s" key. I also added autocomplete which does regex matching with a jQuery plugin against a common index of terms researched from Google Analytics. Results are fetched by an AJAX call to the Google Search Engine API on ENTER; what's more, the results will persist during the session if the user activates the search bar again.

    Zendesk Apps

    zendesk.com/apps | gist | gist

    Creative was swamped so I designed the UI for the single app pages - click "Learn More" on any tile to view. Underneath the hood there is actually a custom template routing plugin I wrote in PHP that dynamically builds the page based on the URI request - these pages are not stored in the MySQL database because the data comes from the Zendesk product! Data is actually stored in a JSON file that we update periodically with a Grunt.js task during development.

    BONUS: CSS/JS Minification and Cache Busting

    In the source code you'll notice hashes and a ".min" suffix on our externally-loaded CSS and JavaScript files. I implemented a set of Grunt.js tasks to minify and hash files, and then make replacements throughout our WordPress templates.
